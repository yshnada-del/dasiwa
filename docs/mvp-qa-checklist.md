# 다시와 MVP QA 체크리스트

## 인증
- [ ] 회원가입 시 이메일, 비밀번호, 샵 이름, 원장님 이름을 입력할 수 있다.
- [ ] 회원가입 성공 후 `profiles`에 프로필이 생성된다.
- [ ] 로그인 성공 후 `/app/dashboard`로 이동한다.
- [ ] 로그아웃 후 `/app/*` 접근 시 `/login`으로 이동한다.

## 고객 관리
- [ ] 고객 이름만으로 고객을 등록할 수 있다.
- [ ] 전화번호, 메모, 선호 시술, 기본 재방문 주기를 함께 저장할 수 있다.
- [ ] 고객 리스트에서 등록한 고객이 보인다.
- [ ] 고객 이름으로 검색할 수 있다.
- [ ] 전화번호로 검색할 수 있다.
- [ ] 고객 카드를 누르면 고객 상세 페이지로 이동한다.
- [ ] 고객 상세에서 기본 정보, 메모, 최근 방문일, 다음 방문 예정일을 확인할 수 있다.

## 시술 기록
- [ ] 고객 상세에서 시술일 기본값이 오늘 날짜로 표시된다.
- [ ] 시술명 필수 입력이 동작한다.
- [ ] 금액을 비우면 `null`로 저장된다.
- [ ] 재방문 주기를 비우거나 기본값으로 저장할 수 있다.
- [ ] 시술 기록 저장 후 리스트에 최신순으로 표시된다.
- [ ] 시술 기록 저장 후 `customers.last_visit_date`가 갱신된다.
- [ ] 시술 기록 저장 후 `customers.next_visit_due_date`가 시술일 + 재방문 주기로 계산된다.

## 사진 업로드
- [ ] jpg, jpeg, png, webp 파일을 업로드할 수 있다.
- [ ] 파일 1개당 5MB 초과 시 에러가 표시된다.
- [ ] 한 시술 기록에 5장 초과 첨부 시 에러가 표시된다.
- [ ] 업로드 후 `treatment_photos.storage_path`가 저장된다.
- [ ] 고객 상세에서 signed URL로 사진 그리드가 표시된다.

## 연락할 고객
- [ ] 다음 방문 예정일이 이번 주 종료일 이하인 고객이 표시된다.
- [ ] 예정일 지남, 오늘 연락 추천, 이번 주 예정 상태가 구분된다.
- [ ] 연락 완료 버튼 클릭 시 `follow_up_logs`에 `done` 기록이 생성된다.
- [ ] 오늘 이미 연락 완료한 고객은 중복 처리되지 않는다.
- [ ] 연락할 고객이 없을 때 빈 상태 안내가 표시된다.

## AI 메시지
- [ ] 최신 시술 기록 기준 후기 요청 메시지를 생성할 수 있다.
- [ ] 연락 고객 카드에서 재방문 유도 메시지를 생성할 수 있다.
- [ ] 생성 중 버튼 중복 클릭이 방지된다.
- [ ] 생성 실패 시 친절한 에러 메시지가 표시된다.
- [ ] 생성된 메시지가 `ai_message_logs`에 저장된다.
- [ ] 복사 버튼으로 메시지를 클립보드에 복사할 수 있다.
- [ ] 복사 성공 시 `copied_at`이 업데이트된다.

## 보안과 계정 분리
- [ ] A 계정의 고객이 B 계정에 보이지 않는다.
- [ ] A 계정의 시술 기록이 B 계정에 보이지 않는다.
- [ ] A 계정의 사진 metadata가 B 계정에 보이지 않는다.
- [ ] private Storage 사진이 signed URL로만 표시된다.
